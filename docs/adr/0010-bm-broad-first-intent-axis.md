# ADR-0010 — BM 방향: broad-first 성장 + "의도(활용 관점)축" 수익화 + 프로슈머 우선 freemium

- 상태: Accepted (가격·한도 수치는 가설 — M2 검증 후 보정)
- 일자: 2026-06-27
- 관련: [GTM](../GTM.md)(§7.2 경계·§8 가격·§6.1 계측), [ROADMAP §4.1](../ROADMAP.md)(전환 트리거), [ADR-0008](./0008-llm-usage-report.md)(활용 관점 리포트=의도축의 원천), [ADR-0011](./0011-llm-report-cache-and-licensing-gate.md)(비용·캐시·약관)
> ⚠️ **보완([ADR-0014](./0014-source-license-segmentation.md))**: 수익화는 네이버/카카오가 아니라 **상업 OK 소스(위키백과 등) 위에서만**(네이버 FAQ: 검색결과 상업 이용·재가공·저장 불가).

## 맥락
서비스 토의에서 두 가지가 드러났다. (1) 데이터가 구조적으로 **공인(연예인·운동선수)으로 수렴**하지만, 오너는 "공인·기업을 포함한 **모든 주제**"를 원하며 처음부터 니치(기업분석·개인정보)로 좁히는 것을 거부한다. (2) BM은 "**큰 틀(broad)에서 시작해 점진적으로 니치 시장으로 좁힌다**". 기존 PRD/GTM은 B2B 기업 리서치를 1순위 비치헤드로 가정했으나 실제 데이터는 공인 우위 → **추측으로 니치를 박으면 틀린 곳에 베팅**하게 된다.

핵심 통찰: 비용(유일 지출=LLM 리포트)·해자(지능 레이어)·의도축(관점 노드)이 **활용 관점 리포트 한 점에서 만난다**(ADR-0008·0011).

## 결정
- **제품(주제)은 영원히 broad** — 주제 제한을 하지 않는다(오너 요구 정합 + 바이럴 표면 최대).
- **BM을 2개 레이어로 분리**: 획득·성장 = 영원히 broad·무료·바이럴 / 수익 = 점진적으로 좁힘. *제품*이 아니라 *"누구에게 무엇을 파는가"*만 좁힌다.
- **좁히는 축 = 주제가 아니라 "활용 관점(의도)"**. LLM 리포트의 관점 노드(투자/취업/경제/…)가 의도축. 어느 의도가 리텐션·지불의향이 높은지 데이터로 찾아 깊게 판다(구글이 주제는 다 열고 "구매 의도" 검색에만 광고를 붙인 구조와 동형). 측정은 GTM §6.1.
- **첫 과금 = 주제 불문 프로슈머 freemium**(니치/B2B는 후행). 게이팅은 **생산·지속·대량 측**(저장·내보내기·모니터링·워터마크 제거·LLM 무제한/심층)에만, **소비(viewer) 측은 전부 무료**(바이럴 보존). 경계표 = GTM §7.2.
- **계정 = 하이브리드**: 검색은 익명(마찰 최소), 저장은 로그인(ADR-0002 익명 MVP를 "저장 시 로그인"으로 확장).
- **해자 = 지능 레이어(관점 리포트 품질) + 저장 워크플로**. 3D는 후킹이지 해자가 아니다.
- **무료 LLM 리포트 = 일 5건 fresh/세션·IP**(초과 시 휴리스틱 폴백) + 7일 리포트 캐시·시드 프리웜 + 출시/바이럴 피크엔 소액($20~50) 스파이크 보험. 비용·캐시 상세 = ADR-0011.
- **가격 가설** = 베타 ₩6,900/월(평생 고정·초기 한정) → 정상 ₩9,900/월, 연 ₩89,000. 전환 목표 M3 2~4%. 검증 = M2 인터뷰(Van Westendorp)·M3 A/B.

## 대안
- **주제 니치 선박기(기업만/공인만)** → 오너 요구(모든 주제)와 충돌 + 바이럴 표면 축소 + 미검증 추측 베팅 → 기각.
- **광고 우선** → 출처·신뢰라는 차별점 훼손 위험 + 트래픽 규모 전 단가 미성립 → 후행·조건부(GTM §7).
- **B2B 인텔리전스 먼저** → 영업 리소스·데이터 신뢰선 필요, 데이터 미검증 → M3+ 후행 트랙.

## 트레이드오프
- (+) 모든 주제 유지로 바이럴·SEO 표면 최대 + 의도 niche를 추측이 아니라 데이터로 발견.
- (+) 비용·해자·의도축이 LLM 리포트 한 점에서 만나 설계가 단순해진다.
- (−) horizontal 제품은 answer engine(Perplexity/ChatGPT)과 broad 가치가 겹친다 → "출처 붙은 3D 지도 + 활용 관점"의 좁은 가치에 포지션 집중(오버포지셔닝 금지).
- (−) "인기 많지만 수익 0" 함정 위험 → Phase 0 계측(GTM §6.1)으로 의도축의 지불의향·리텐션을 조기 측정, 전환 트리거(ROADMAP §4.1) 재사용으로 좁힐 시점 판단.
- (−) 무료 LLM 노출은 비용 천장과 긴장 → ADR-0011의 2단 캐시·일 상한·서킷으로 방어.
