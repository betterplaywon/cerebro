# ADR-0014 — 소스 라이선스 분리: 무료 표시(네이버·카카오) vs 수익화·재가공(상업 OK 소스)

- 상태: Accepted (네이버 FAQ·약관 오너 확인으로 확정 · 구현 재정렬은 점진 M2/M3)
- 일자: 2026-06-27
- 보완 대상: [ADR-0010](./0010-bm-broad-first-intent-axis.md)(BM)·[ADR-0011](./0011-llm-report-cache-and-licensing-gate.md)(캐시)·[ADR-0012](./0012-monitoring-alerts.md)(모니터링)의 암묵적 네이버/카카오 가정. 근거: 네이버 오픈API FAQ + [DATA-SOURCING §11](../DATA-SOURCING.md)

## 맥락
네이버 오픈API FAQ/약관(오너 직접 확인, 2025-05-27 개정) 결과:
1. **상업적 이용 불가** — 배너광고·스폰서 또는 **유료회원 전용** 접근이면 상업적 이용. "검색결과를 보여주는 서비스 / 유사 검색서비스"는 검색결과를 **상업적으로 사용 불가**.
2. **저장·재가공 불가** — 부하용 **단순 캐시**만 허용.
3. 무료 제공.

카카오도 유사 제약(사전승낙 없는 복제·장기보관·캐시 제한, ADR-0011). 즉 **네이버·카카오 검색결과로는 수익화·재가공·장기저장을 할 수 없다.** 반면 **위키백과(CC BY-SA)는 상업·파생·저장을 허용**(출처·동일조건). 그동안 cerebro의 수익화·재가공 레이어(LLM 리포트·7일 캐시·저장 보드·모니터링)는 네이버/카카오를 암묵적으로 포함했다 → 토대 재정렬 필요.

## 결정
소스를 **라이선스 기준 2개 레이어로 분리**한다.
- **Layer A — 무료 표시·단순캐시(네이버·카카오)**: 검색결과를 **출처 표기해 독립 노드로 표시**(약관 2.1 "독립적 노출·무수정") + **부하용 단순 캐시(≤30분)만**. **재가공(LLM 입력)·장기저장(7일 캐시·저장 보드·모니터링 지문)·수익화(유료/광고) 금지.**
- **Layer B — 수익화·재가공·저장·모니터링(상업 OK 소스만)**: **위키백과(CC BY-SA, 출처·동일조건·DRM 금지)** 앵커 + 공공데이터포털 + 앱스토어(iTunes) + 유튜브(Data API) + **상업용 웹검색(Tavily 유료 등)**. LLM 활용 리포트·7일 리포트 캐시·저장 보드·모니터링은 **이 레이어 위에서만**.
- 수집 파이프라인은 **"표시 소스"와 "분석/저장 소스"를 분리**(수집 단계 라벨링). **무료 LLM 리포트도 Layer B 입력만** 사용(무료/유료 일관).

## 대안
- **네이버 1:1 문의로 예외 타진** → 시간·불확실. 보수적 분리를 기본으로 두고 필요 시 병행.
- **네이버 상업 제휴** → 인디 규모에 과중 + "검색서비스 상업 불가" 기조상 난망 → 기각.
- **무료 단계는 네이버 재가공 유지(관용 해석)** → "재가공 불가"와 충돌·BM 토대 오염 위험 → 보수적 분리 채택.

## 트레이드오프
- (+) 수익화·재가공·저장이 라이선스상 안전한 토대(위키+상업 소스) 위에 선다. 네이버/카카오 무료 한국어 커버리지는 표시 레이어로 유지.
- (−) **무료 LLM 리포트의 한국어/시의성 깊이 약화**(네이버/카카오를 분석 입력에서 제외) → 보강: 상업용 웹검색(Tavily 유료)·공공데이터를 Layer B에 도입. **[ADR-0005](./0005-defer-broad-web-search.md)(웹검색 보류)의 부분 해제 트리거 = "분석/Pro 레이어용 상업 웹검색".**
- (−) 수집 파이프라인에 소스 레이어 분리 구현 필요(표시 vs 분석/저장) → M2/M3 재정렬(현 무료 토이는 저위험 → 점진 적용).
- (−) 카카오도 Layer A → 커뮤니티(cafe/kin) 신호도 분석 레이어에서 제외.

## 영향
- ADR-0010: 수익화는 Layer B 소스 위에서만. ADR-0011: 7일 리포트 캐시·프리웜은 Layer B 산출물만(네이버/카카오는 30분 단순 캐시 유지). ADR-0012: 모니터링은 Layer B 소스만. DATA-SOURCING §11 갱신.
