# ADR-0011 — LLM 리포트 비용 방어: 2단 캐시(데이터 30분 / 리포트 7일) + 시드 프리웜 + 상업 이용 약관 게이트

- 상태: Accepted (무료 단계 즉시 적용 · 구현 = `feat/llm-report-cache-prewarm` · 유료화 게이트는 M3)
- 일자: 2026-06-27
- 관련: [ADR-0008](./0008-llm-usage-report.md)(LLM 리포트·예산 $8.8), [ADR-0010](./0010-bm-broad-first-intent-axis.md)(BM 방향), [DATA-SOURCING §11](../DATA-SOURCING.md)(라이선스 게이트), [BACKLOG](../BACKLOG.md)(NOW#3 서킷 브레이커)
> ⚠️ **보완([ADR-0014](./0014-source-license-segmentation.md))**: 7일 리포트 캐시·프리웜은 **상업 OK 소스(Layer B) 산출물만**. 네이버/카카오 검색 데이터는 30분 단순 캐시만(재가공·장기저장 불가).

## 맥락
비용이 드는 **유일한 지점은 LLM 활용 리포트**(검색당 $0.03~0.05). 그래프 데이터(공식 API 수집)는 쿼터만 소모(무료). 현재 캐시는 **전체 GraphSnapshot(데이터+리포트)을 30분 단일 캐시**(`CACHE_TTL_MS`)로 묶는다(`search-orchestrator.ts`) → 인기 주제도 30분마다 LLM 재호출. 예산 $8.8 ≈ fresh 175~290건으로 너무 작아 무료 LLM을 의미 있게 서빙하기 어렵다(ADR-0008).

동시에 약관 리서치 결과: **카카오** 약관(이용약관 제11조3항·운영정책 제5조30호)은 API 데이터의 사전승낙 없는 복제·장기보관·제3자 제공을 제한하고, 캐시는 "UX 목적 + 최신 유지"일 때만 허용(제5조20호) → **장기 스냅샷 보관은 위반 소지, 30분 갱신 캐시는 정합**. **네이버** 약관 원문은 접근 차단 → 오너 직접 확인 게이트(DATA-SOURCING §11).

## 결정
- **2단 캐시로 분리한다.**
  1. **소스 데이터(스니펫·그래프) = 단기 갱신 캐시(30분, `CACHE_TTL_MS` 유지)** → 신선도·API 쿼터 방어 + 카카오 "갱신되는 캐시" 정합.
  2. **LLM 리포트(우리 2차 분석 산출물) = 장기 캐시(기본 7일, 신규 `REPORT_CACHE_TTL_MS`)** → 비용 방어. 리포트 캐시 적중 시 `analyzeUsage` 스킵(LLM 미호출). 리포트의 `sourceIds`가 새 수집과 불일치하면 해당 링크만 무시(graceful degrade).
- **시드 프리웜**: 대표 시드 코퍼스(기업·공인 등)의 리포트를 미리/주기적으로 캐시 → 데모·공유 최다 주제는 즉시·$0, 주기 갱신으로 신선도 유지.
- **글로벌 서킷 브레이커**(BACKLOG NOW#3): 월 추정비용이 상한(기본 $8) 도달 시 전체 휴리스틱 폴백. 무료 **일 5건/세션·IP** 상한과 병행.
- **저장 보드(Pro)는 스니펫 영구저장 금지** — 쿼리 + 그래프 구조 + 출처 링크만 저장, 열람 시 라이브 재조회(약관·신선도·PIPA 정합). 상세 = DATA-SOURCING §11.

## 대안
- **전체 스냅샷 7일 캐시** → 가장 단순하나 카카오/네이버 데이터를 7일 보관 = 약관 충돌 + stale → 기각.
- **30분 유지 + 비용은 서킷 상한만** → 인기 주제 LLM 반복 호출로 예산 빠르게 소진 → 2단 캐시로 보완.
- **관점별 다중 호출·캐시** → 복잡·이득 작음 → 리포트 1건 단위 캐시 채택(ADR-0008의 1회 호출 유지).

## 트레이드오프
- (+) 비용(LLM)·신선도(데이터)·약관(카카오 갱신캐시)을 동시 충족. 인기 주제 LLM 호출이 30분→7일당 1회로 격감.
- (+) 데이터는 30분마다 재수집 → API 쿼터 사용↑(무료, 네이버 25k·카카오 50k/일 한도 내) 대신 신선·합법.
- (−) 리포트가 최대 7일 된 분석일 수 있음 → 휘발성 주제(뉴스·주가)는 stale. 완화: 추후 주제 휘발성별 TTL 차등 + Pro "재생성".
- (−) `sourceId` 불일치 시 일부 출처 링크 누락(graceful degrade로 흡수).
- (−) 네이버 약관 미확인 → 유료화 전 오너 확인 게이트(DATA-SOURCING §11).
