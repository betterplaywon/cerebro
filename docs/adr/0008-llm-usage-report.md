# ADR-0008 — 활용 관점 리포트: 수집 정보를 Claude로 정제해 "정보 활용 방법"을 제공

- 상태: Accepted
- 일자: 2026-06-26
- 관련: [ADR-0001](./0001-graph-contract-ssot.md)(그래프 계약 SSOT), [ADR-0006](./0006-node-category-palette.md)(노드 팔레트)

## 맥락
기존 마인드맵은 중심 노드에서 카테고리/토픽 노드로 가지를 뻗고, 각 노드는 출처 링크만
제공했다. 사용자 피드백: **"출처에서 메인 정보를 정리하고, 그 정보를 어떻게 활용할 수 있는지
(예: 주식 호재/악재, 취업에 도움, 국내 경제 타격)를 단문이 아니라 하나의 리포트처럼 보여달라."**

즉 요구는 두 가지다.
1. 수집 출처에서 **핵심 정보 요약**(사실 정리).
2. 그 정보의 **활용 관점별 상세 리포트** — 주체마다 다른 호재/악재·기회·리스크 분석.

(2)는 주체(기업/제품/공인)마다 내용이 완전히 달라지는 **개방형 서술**이다. 빈도·규칙 기반
휴리스틱(현 토픽 추출)으로는 "이 정보가 투자에 호재인 이유"를 쓸 수 없다 — 의미 이해가 필요하다.
따라서 **LLM 도입이 사실상 불가피**하다(룰 기반 대안 없음).

## 결정
검색당 1회 **Claude Sonnet 4.6** 호출로 `{summary, angles[]}`를 생성하고, 이를 그래프로 만든다.
- **중심 노드** = 핵심 정보 요약(`report`).
- **활용 관점 노드(`usage`)** = 관점별 상세 리포트. 중심에서 `relation:'활용'` 엣지로 가지.
- 후보 관점 9종(주체에 **해당되는 것만** LLM이 선택): 투자·취업/커리어·경제/산업·사회·건강·
  관계/접점·쇼핑·도서·콘텐츠.
- 계약 확장(`packages/shared`): `NodeKind`에 `'usage'` 추가, `GraphNode.report?`(서술형 본문) 추가.
- 출력 구조화: `output_config.format`(json_schema) + zod 런타임 검증(`AnalysisSchema`). 관점→출처
  매핑은 LLM이 반환한 출처 번호(`sourceRefs`)를 실제 `Source.id`로 변환(존재하는 것만).

### 비용 통제 (예산 $8.8)
- **키 게이트**: `ANTHROPIC_API_KEY` 미설정 시 분석 비활성 → 기존 휴리스틱 그래프로 폴백(**지출 0**).
- **검색당 1회 호출**, thinking 미사용, 상위 출처 **18건만** 전송(`MAX_SOURCES`), `max_tokens=4000`.
- 결과는 기존 **검색 캐시(30분)**로 재사용 → 동일 쿼리 재요청은 추가 지출 0.
- 추정 단가: 입력 ~3k·출력 ~2.5k 토큰 ≈ **검색당 $0.03~0.05** → $8.8로 **약 175~290건의 고유 검색**.
- 모델: 품질·비용 균형으로 **Sonnet 4.6**($3/$15 per 1M). Opus(4배 비쌈)는 과투자, Haiku는
  서술 품질 부족으로 기각.

### PIPA 가드
- 개인은 **공인·공개정보 한정**, 비공개 개인 신상 프로파일링 금지, 민감정보(주민번호·연락처·
  주소·건강·금융·정치/종교/성적지향) 생성·추론 금지 — 시스템 프롬프트에 명시.
- 투자 관점 리포트는 끝에 `(투자 조언이 아님)`을 덧붙여 책임 경계 표시.

## 대안
- **룰 기반 활용법**(고정 문구) → 주체별 호재/악재를 표현 불가, 요구의 핵심을 충족 못 함 → 기각.
- **Opus 4.8** → 비용 4배($5/$25), $8.8 예산에 과투자 → 기각(필요 시 모델만 교체 가능).
- **항상 호출(키 강제)** → 키 없는 환경/CI에서 지출·실패 → 게이트로 폴백 채택.
- **여러 번 호출(관점별 1회)** → 비용 N배 → 1회 호출로 전체 생성 채택.
- `zodOutputFormat` 헬퍼 → SDK가 `zod/v4`를 요구(프로젝트는 zod v3) → raw json_schema +
  수동 zod v3 검증으로 우회.

## 트레이드오프
- (+) 요구 정확 충족: 출처 요약 + 주체 맞춤 활용 리포트(단문 아님). 관점이 자식 노드로 펼쳐짐.
- (+) 키 없으면 0원·자동 폴백 → 무키 개발/CI 안전, 점진 도입 가능.
- (+) 모델·관점·상한이 모두 1곳(`analyze/report.ts`, `env.ANALYSIS_MODEL`)에 격리 → 교체 쉬움.
- (-) LLM 의존 = 유료·지연·환각 가능성. 완화: 출처 근거 강제, 18건 상한, 캐시, 폴백, "추정" 표기 지시.
- (-) 활용 리포트는 LLM 판단 → 사실성은 출처에 의존(과장·단정 금지 지시로 best-effort).
- (-) 예산 소진 시 분석 중단 필요 → 현재는 키 제거로 수동 차단(자동 상한·사용량 메트릭은 후속).
