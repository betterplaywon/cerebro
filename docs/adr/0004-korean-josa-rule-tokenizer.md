# ADR-0004 — 한국어 조사 분리는 규칙 기반으로 (형태소 분석기 도입 보류)

- 상태: Accepted
- 일자: 2026-06-26

## 맥락
토픽 추출(`apps/api/src/collect`)은 제목·스니펫을 `tokenize()`로 잘라 토큰 빈도로 가지(concept) 노드를 만든다.
기존 토큰화는 `[^0-9a-z가-힣]+` 분리만 해서 **조사가 어간에 붙은 채** 남았다.
- `토스가`·`토스는`·`토스를`·`토스의`가 `토스`와 다른 토큰이 되어 **같은 개체의 빈도가 파편화**됨 → 토픽이 약해지고 중복.
- `대한민국의`가 `대한민국`에 합류하지 못함.

단순 규칙으로 조사를 떼면 **오절단** 위험이 있다: `음악가`→`음악`, `국가`→`국`, `물가`, `대학가`, `누군가`처럼
말음이 조사와 동음인 실제 단어를 깨뜨린다.

## 결정
**의존성 0 규칙 기반 조사 분리**를 도입한다(`apps/api/src/collect/korean.ts`의 `stripParticle`). 형태소 분석기는 **보류**.

핵심 안전장치는 한국어 조사의 **받침(종성) 이형태 규칙**이다:
- 모음형 조사(`가`/`는`/`를`/`와`)는 **모음으로 끝난 어간** 뒤에만,
- 자음형 조사(`이`/`은`/`을`/`과`)는 **자음으로 끝난 어간** 뒤에만 결합한다.

이 규칙은 `(음절코드 − 0xAC00) % 28`로 종성을 계산해 판정하며, `음악가`·`국가`·`물가`·`대학가`·`누군가`
(모두 받침 뒤 `가`)를 **사전 없이 결정적으로** 차단한다. 받침으로 못 거르는 잔여 오탐은 세 가드로 막는다:
- **최소 어간 길이 2음절** (`국가`→`국`, `평가`→`평` 등 차단),
- **구조적 `주의(主義)` 가드**: `의`를 떼는 경우 어간이 `주`로 끝나면 절단하지 않는다 →
  `민주주의`·`자본주의`·`개인주의`·`권위주의`… **개방형 ism 전체를 한 줄로** 차단(`州` 지명+의 소수 미절단은 감수),
- **큐레이션 보호 단어 사전**(`여의도`·`충무로`·`다나와`·`산부인과`·`떡볶이`·`처음처럼` 등 — 받침·길이 가드를 통과하지만
  절단하면 안 되는, 자주 검색되는 고가치 단어).

보호 사전은 **적대적 코퍼스 테스트**로 발견한 실사용 오탐(지명·역명·브랜드·진료과·받침+이 보통명사)을 회귀로 고정한다.
가장 긴 조사를 먼저, **한 번만** 절단한다(재귀 절단에 의한 과침식 방지). `tokenize()`와 검색어 토큰이 같은
규칙을 거치므로 중심-가지 중복 제외도 일관되게 동작한다.

## 대안
- **형태소 분석기 도입**
  - `kiwi-nlp`(Kiwi, wasm): 진짜 형태소 분석·품사 필터·원형 복원으로 파편화를 근본 해결. 단 **~4MB wasm + 기동 시 모델 로드**, LGPL-2.1. → MVP에는 과함. **유일한 합리적 업그레이드 경로로 보류(defer)**.
  - `mecab-ko`/`mecab-ya`: 네이티브 MeCab + 사전 설치 필요 → 배포 단순성 훼손, 2022년 이후 정체. **회피**.
  - `koalanlp`: JVM 의존 → 최중량. **회피**.
  - `es-hangul`: 가볍고 MIT지만 **조사 분리가 아니라 조사 선택(이/가 부착)·자모 분해** 도구라 용도 불일치. **부적합**.
  - `natural`: 한국어 형태소 미지원(Porter 등 서구 스테머). **부적합**.
- **집계 단계 증거 기반 병합**(`extractTopics`에서 `어간`이 코퍼스에 독립 등장할 때만 병합): 오탐이 더 적지만
  `대한민국의` 단독 사례를 못 고치고 집계 레이어 복잡도 상승. → 현 단계 보류, 필요 시 후속.

## 트레이드오프
- **장점**: 의존성·네이티브 바이너리·wasm 없음(배포 단순성 유지, YAGNI 부합), 받침 규칙은 블랙리스트가 아닌
  원리적 가드라 오탐 클래스를 구조적으로 제거, 변경 범위가 `korean.ts` + `tokenize` 2줄로 국소적.
- **비용/한계**: 적대적 검증 결과, 받침·길이 가드는 **회사·인물·제품명에는 강했지만** 개방 클래스 — 도(島)/로(路)
  지명·역명, 받침+`이` 보통명사(`떡볶이`·`곰팡이`…), `~과(科)` 진료과 — 에는 보호가 약하다. `주의` 계열은 구조적
  가드로 해결했지만 나머지는 보호 단어 사전에 의존하므로 **사전은 비완전**이며 신규 고가치어는 추가가 필요하다(유지보수).
  품사 문맥이 없어 일부 부사·활용형 말음 오인 가능, 복합명사 분해·원형 복원은 다루지 않음. 토픽 출력이 바뀌므로
  테스트 갱신 필요. 개방 클래스가 토픽 품질을 떨어뜨리면 §대안의 **증거 기반 병합** 또는 `kiwi-nlp` 보류 결정을 재검토.
- **회귀 안전장치**: `korean.test.ts`가 true-positive/false-positive(오절단)/passthrough 코퍼스를 고정. 새 오탐 발견 시
  보호 사전에 추가. 토픽 품질이 규칙 한계에 부딪히면 `kiwi-nlp` 보류 결정을 재검토한다.
