# ADR-0012 — 모니터링/알림: 주제+선택 의도 구독 · 지문 기반 변화감지 · 이메일+인앱 (Pro)

- 상태: Accepted (설계 확정 · cyber-security 적대적 검증 H1~H4 반영 · 구현 M2/M3 — 하이브리드 계정 + **비용 서킷/서브캡 선행** · 구현 직전 보안 재검증 게이트)
- 일자: 2026-06-27
- 관련: [FEATURE-MONITORING](../FEATURE-MONITORING.md)(상세 스펙), [ADR-0010](./0010-bm-broad-first-intent-axis.md)(의도축 BM), [ADR-0011](./0011-llm-report-cache-and-licensing-gate.md)(캐시·약관), [ADR-0008](./0008-llm-usage-report.md)(LLM 리포트), [DATA-SOURCING §11](../DATA-SOURCING.md)(라이선스 게이트), [GTM §7.2](../GTM.md)(경계)

## 맥락
BM에서 모니터링/알림을 **"의도축 정점 레버"**(일회성 검색 → 지속 의도 = 리텐션·해자, 고지불의향 의도=투자·취업·평판)로 지목했다(ADR-0010, GTM §7.2). 설계 제약: ① 카카오/네이버 약관상 API 스니펫 장기저장 금지(ADR-0011) ② PIPA(지속 모니터링=프로파일링 우려) ③ 비용(주기적 LLM 호출은 구독수×체크수로 폭증) ④ 계정 선행(하이브리드 계정 M2).

## 결정
- **구독 단위 = 주제 + 선택 의도(활용 관점)**. 기본 주제, 의도 focus는 옵션. Pro 전용·활성 구독 한도(무료=0).
- **변화 감지 = "지문"만 저장**(출처 URL 해시·콘텐츠 해시·직전 리포트 요약 해시) — 스니펫 원문 미저장(약관). 매 체크 fresh 재수집 → diff → **새 출처 감지 시에만 LLM 1회**. 단 상태·재수집·LLM은 **주제(+의도) 단위 공유**(구독자에 fan-out — 쿼터·비용 dedup), 빈결과/오류는 **fail-closed**(거짓 알림 방지).
- **전달 = 이메일(재유입 루프) + 인앱 피드**. 카카오 알림톡·근실시간=후속.
- **가드(cyber-security 적대적 검증 반영)** = 비용: 전역 서킷+월 예산캡(**현재 미구현 — 선행 전제**)+모니터링 서브캡, 주제 dedup+쿼터 예산(≤30%)+admission control / PIPA: 구독 생성 공인 게이트·주기 재검증·민감 이벤트 fail-closed 필터·보존기간 캡·삭제요청 cascade(구독자 해지와 별개) / 보안: HMAC 1회성 해지 토큰·POST·RLS(IDOR 차단)·알림 출력 인코딩 / 약관: 지문만·매회 갱신·요약 추상화. 상세 = FEATURE-MONITORING §4~7·§13.
- **단계**: MVP(주제·일1회·새출처 트리거·이메일+인앱) → 확장(의도 정밀·주기 선택·중요도 스코어링·알림톡·다이제스트). 전제 = 하이브리드 계정(M2).

## 대안
- **매 체크 LLM 전체 재분석** → 정확하나 비용이 구독×체크로 폭증 → 기각(새 출처 트리거로 한정).
- **스니펫 저장 후 텍스트 diff** → 카카오/네이버 약관 위반 → 지문만으로 대체.
- **주제만 구독(의도 무관)** → 노이즈↑·의도축 연결 약함 → 주제+선택 의도.
- **인앱만** → 재유입(리텐션) 루프 약함 → 이메일 추가.
- **구독 단위 state(주제 중복 재수집)** → 인기 주제는 구독자 수만큼 쿼터·비용 곱증 → **주제 단위 공유 state**로.

## 트레이드오프
- (+) 의도축 레버·리텐션·고WTP 정렬. 비용은 변화 시에만 발생. 약관·PIPA 정합.
- (−) 새 출처 없이 "내용만 바뀐" 변화는 놓칠 수 있음(지문 한계) → 변화 중요도 스코어링은 후속.
- (−) 지속 모니터링은 PIPA 민감 → 공인 한정·즉시 삭제·민감정보 추론 금지로 완화. **구현 전 보안 검증 필수.**
- (−) 이메일 provider·스케줄러 = 신규 인프라(소액 비용) → M2/M3 단계 비용으로 편입.
- (−) 비용 가드(서킷·예산캡)가 **선행 미구현** → 모니터링 출시는 그 구현([BACKLOG](../BACKLOG.md) NOW#3)에 종속.
- (−) 주제 단위 공유 state는 비용·쿼터를 막지만 사용자별 미세 맞춤(개인 주기 등)을 제약 → 수용.
