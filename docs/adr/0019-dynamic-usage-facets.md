# ADR-0019 — 활용 관점을 고정 9개 enum에서 주제 동적 생성으로 전환

- 상태: Accepted (오너 요청 — "카테고리가 한정적, 훨씬 다양한 관점 필요")
- 일자: 2026-06-28
- 보완 대상: [ADR-0008](./0008-llm-usage-report.md)(LLM 활용 리포트)의 관점(angle) 산출 방식. ADR-0011(캐시)·ADR-0013(예산)·ADR-0014/0018(레이어 게이트) 전제는 유지.

## 맥락
활용 관점 노드(`kind: 'usage'`, 전구 아이콘)는 LLM이 생성하지만, `report.ts`의 **고정 9개 enum**(`investment·career·economy·society·health·relationship·shopping·books·content`)에서만 고르도록 출력 스키마(`ANALYSIS_JSON_SCHEMA`의 `key.enum`)로 강제돼 있었다.

결과: 어떤 주제를 검색해도 동일한 범용 버킷의 부분집합만 나온다. `nvidia`를 검색해도 "투자 관점·취업·커리어·경제·산업·쇼핑·콘텐츠·사회"처럼 주제 무관한 일반 카테고리만 보이고, "AI·데이터센터", "GPU 기술", "경쟁 구도", "수출 규제 리스크" 같은 **주제 밀착 관점이 구조적으로 불가능**했다. 마인드맵의 핵심 가치(주제를 여러 각도로 펼쳐 보기)가 9칸 틀에 갇혀 있었다.

## 결정
고정 enum 제약을 제거하고, **LLM이 주제·출처에 맞춰 관점 라벨을 직접 생성**한다.

- 출력 스키마의 `angles[].key`(enum) → `angles[].label`(자유 문자열)로 교체. 관점은 목록에서 고르는 게 아니라 주체에 맞춰 새로 만든다.
- 프롬프트는 다축(핵심 사업·기술·경쟁·실적·규제·지정학·역사·사회영향·논란·트렌드·투자·취업·소비경험 등)을 **예시로만** 제시하고, 주제에 밀착된 라벨을 서로 구별되게 만들도록 지시. 풍부한 주체는 세부 축까지 펼쳐 10개 이상으로(최대 16개) 풍성하게.
- 노드 id는 `usage-<key>` → `usage-<index>`(키가 사라졌으므로 스냅샷 내 인덱스로 유일·안정).
- 가드 보존: 투자·주가류 관점은 report 끝에 "(투자 조언이 아님)", PIPA(공인·공개정보 한정·민감정보 금지), 출력측 PII 재마스킹(`redactSensitive`)을 **라벨에도** 적용. 라벨은 길이 상한·공백 정규화·중복 제거로 정제.
- 관점 가지 상한 = 휴리스틱(MAX_BRANCHES=10)과 분리한 `MAX_USAGE_BRANCHES=16`. 3D 레이아웃은 가지 수 √비례로 구 반경을 키워 밀집 방지.
- 출력 토큰 4000→8000, 입력 출처 18→30으로 상향(관점 수↑에 따른 절단 방지 + 다양성 재료 확보).

shared 계약은 무변경 — `GraphNode.label`은 원래 자유 문자열, `kind:'usage'`도 그대로. 제약은 오직 `report.ts` 출력 스키마에만 있었으므로 변경은 `apps/api`에 국한된다.

## 대안
- **고정 목록 확장(9→20개)**: 단순하고 결정론적이나 여전히 범용·고정 틀. "AI·데이터센터"처럼 주제 고유 관점은 못 담음 → 다양성 요구를 근본적으로 못 푼다. 기각.
- **하이브리드(고정 앵커 N개 + 동적 M개)**: 일부 일관성은 얻지만 복잡도↑, 앵커가 다시 범용 버킷을 강제. 동적이 다양성을 직접 달성하므로 불필요(YAGNI).
- **클라이언트(web) 동적 라벨 대응 필요?** 아니오 — web은 `kind`로 색·아이콘, `label`은 그대로 렌더 → 변경 불필요.

## 트레이드오프
- (+) 주제마다 고유하고 다양한 관점 노드. 마인드맵 탐색 가치 회복(오너 요구 직접 충족).
- (+) 변경 범위 좁음(백엔드 1파일 + 테스트). 가드(투자 면책·PIPA·재마스킹) 전부 보존.
- (−) 라벨 결정론 상실: 같은 주제라도 호출마다 라벨 표현이 달라질 수 있음. 완화 = 7일 리포트 캐시(ADR-0011)로 동일 주제 재현, 길이·중복 정제.
- (−) 토큰·비용 소폭↑(출력 4000→8000, 출처 18→30). 출력 토큰은 LLM이 실제로 낸 관점 수에 비례하므로 빈약한 주체는 거의 안 늘고, 풍부한 주체만 상승. 완화 = 키 게이트·7일 캐시·예산 서킷(ADR-0013)이 상한을 유지.
- (−) enum 생성단계 강제(드리프트 잠금 테스트)의 보호가 라벨엔 적용 불가 → 프롬프트 가드 + 출력측 정제로 대체.

## 영향
- `report.schema.test.ts`: enum 정합 잠금 제거, 라벨 자유문자열 검증으로 대체(드리프트 잠금은 키집합·타입·required·additionalProperties로 유지).
- ADR-0008: 관점 산출 방식만 갱신(중심-가지 구조·리포트·인용·캐시 전제 불변).
- 후속(선택): web 범례 문구를 "활용 관점"에서 더 포괄적으로 다듬을 수 있음(이번 범위 외).
