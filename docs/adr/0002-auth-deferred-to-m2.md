# ADR-0002 — 인증은 M2로 연기 (MVP는 익명)

- 상태: Accepted
- 일자: 2026-06-25

## 맥락
PRD(§4.2)·ARCHITECTURE는 MVP를 **익명 단발 검색**(로그인 없음)으로 정의하지만, SECURITY.md §6은 Supabase Auth/JWT 검증을 MVP에 존재하는 것처럼 기술해 시점이 어긋났다.

## 결정
- **M1(MVP)**: 인증 없음(익명, 읽기 중심). 남용 방지는 **rate limit + 캐시 + (옵션)경량 토큰**으로 처리.
- **M2부터**: Supabase Auth(이메일/OAuth) 도입, 매 요청 JWT 검증, 개인화/저장 기능.
- **즉시 적용(예외)**: Supabase를 쓰는 순간부터 **RLS 활성**과 **service_role 키 서버 전용**은 무조건 지킨다.

SECURITY.md §6 상단에 적용 시점 주석을 추가해 정합성을 맞춘다.

## 대안
MVP부터 인증 도입: 가입 마찰로 초기 유입·바이럴(GTM)에 불리, 무료 운영 복잡도 ↑ → 기각.

## 트레이드오프
익명 MVP는 개인화·남용추적이 약하다. rate limit·캐시로 1차 방어하고, 신호(트래픽·남용)가 임계치를 넘으면 M2 인증을 앞당긴다.
