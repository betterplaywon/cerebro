# ADR-0018 — 개인 전용 모드: 비공개·비영리 인스턴스에서 Layer A를 LLM 리포트에 재포함

- 상태: Accepted (오너 결정 — 개인 전용 운영 전제)
- 일자: 2026-06-28
- 보완/완화 대상: [ADR-0014](./0014-source-license-segmentation.md)(소스 라이선스 분리)의 단일 게이트. ADR-0008(LLM 리포트)·ADR-0011(캐시) 전제 유지.

## 맥락
[ADR-0014](./0014-source-license-segmentation.md) LAYER-SPLIT 이후 LLM 활용 리포트 입력이 **Layer B(상업 OK 소스)로 한정**되었다. 네이버·카카오(Layer A)는 약관상 검색결과의 **재가공·장기저장 금지**(부하용 단순 캐시 ≤30분만 허용)라 LLM 입력에서 제외했기 때문이다.

그 결과 무료로 활성화된 Layer B 소스가 사실상 **위키백과 하나뿐**(공공데이터·Tavily 미활성)이 되어:
- 위키 문서가 없는 주제 → `analyzeUsage`가 `null` → **활용 리포트가 통째로 사라짐**(휴리스틱 폴백).
- 위키 문서가 있어도 입력이 ~8건으로 얇아 **한국어·시의성 깊이 약화**.

cerebro의 핵심 가치(정보 활용 리포트)가 위키 유무에 좌우되는 공백 상태. 오너는 두 제약을 모두 인지한 상태에서 다음을 확인했다:
- **수익화 포기로는 풀리지 않음**: 약관 제약은 "상업적 이용"과 "재가공·저장"이 **별개**이고, LLM 리포트를 막는 건 후자(재가공)다. 비영리라도 재가공은 금지.
- **7일 캐시 제거로도 풀리지 않음**: "저장·재가공" 중 캐시는 저장에만 해당, LLM 호출 자체가 재가공이라 남는다.

## 결정
**운영 형태를 "운영자 본인만 쓰는 비공개·비영리 인스턴스(개인 전용)"로 스코프를 좁히는 것을 오너가 선택**하고, 이 전제에서 Layer A(네이버·카카오)를 LLM 활용 리포트 입력·인용·캐시에 다시 포함한다. 이는 "검색결과를 보여주는 **서비스를 타인에게 제공**"하는 행위가 아니라 운영자 개인이 공개정보를 자기 용도로 조회·정리하는 사용에 해당한다는 판단이다.

구현은 **env 플래그 `PERSONAL_USE_MODE`(기본 OFF)** 로 게이트한다.
- `report.ts`의 ADR-0014 단일 게이트 한 줄을 분기: `PERSONAL_USE_MODE ? items : items.filter(layer==='B')`. 한 줄이라 LLM 입력·인용(sourceIds)·7일 리포트 캐시가 함께 정합(개인 모드=A+B, 공개 모드=B만).
- `env.ts`에서 zod `booleanFlag`로 검증, 기본 false(부팅 경계).
- 코드 기본값은 **ADR-0014 게이트 유지**(공개 안전). 완화는 운영자가 로컬 `.env`에서 명시적으로 켤 때만 적용.

## 대안
- **공개 서비스 유지 + Layer B 보강(공공데이터·Tavily)**: 약관상 가장 안전하나 보강 소스 도입 전까지 공백 지속. 후속으로 병행(개인 모드와 배타 아님).
- **게이트 영구 제거(플래그 없이)**: 레포가 **PUBLIC**이라 기본 동작이 약관 위반 상태로 공개됨 → 기각. 플래그 기본 OFF로 공개 기본값을 안전하게 유지.
- **약관 1:1 문의로 개인용 예외 확인**: 시간·불확실. 개인 전용 전제로 진행하되 공개 전환 시 재확인.

## 트레이드오프
- (+) 개인 전용 인스턴스에서 '처음처럼' 한국어/시의성 깊은 활용 리포트 복원. 핵심 가치 회복.
- (+) 플래그 기본 OFF + PUBLIC 레포 기본값 = ADR-0014 컴플라이언스 유지(되돌릴 수 있는 opt-in).
- (−) **공개 전환 비가역 제약**: 이 인스턴스를 **타인에게 제공·공개 배포·수익화하려면 반드시 `PERSONAL_USE_MODE`를 끄고** Layer B 보강으로 전환해야 한다. 개인 모드로 생성·캐시된 Layer A 파생 리포트는 공개 전 폐기.
- (−) 약관 해석 리스크(수락): "개인용은 서비스 제공이 아니다"라는 해석에 의존. 오너 책임 하 결정. 공개 시 ADR-0014 게이트 복귀가 안전판.

## 영향
- ADR-0014: 단일 게이트는 유지하되 `PERSONAL_USE_MODE`로 분기(공개=B만). ADR-0010(수익화)·ADR-0012(모니터링)은 **공개·수익화 경로에서만** 유효 → 개인 모드와 동시 활성 금지(개인 모드는 비영리 전제).
- 운영: 공개/배포 체크리스트에 "`PERSONAL_USE_MODE` OFF 확인 + Layer A 파생 캐시 폐기"를 하드게이트로 추가(후속 DEPLOYMENT 반영).
