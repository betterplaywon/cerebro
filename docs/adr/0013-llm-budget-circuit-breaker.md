# ADR-0013 — LLM 예산 자동 상한(서킷 브레이커)

- 상태: Accepted
- 일자: 2026-06-27
- 관련: [ADR-0008](./0008-llm-usage-report.md)(활용 리포트·예산 $8.8), [ADR-0011](./0011-llm-report-cache-and-licensing-gate.md)(2단 캐시), [ADR-0012](./0012-monitoring-alerts.md)(모니터링 — 비용 가드 선행 전제), [BACKLOG NOW#3](../BACKLOG.md)

## 맥락
활용 리포트(ADR-0008)는 키 게이트·캐시·폴백으로 비용을 통제하지만, "지출 상한"의 유일한 방어가
**수동 키 제거**뿐이었다(ADR-0008 트레이드오프 말미). 실트래픽·배포 시 사고 한 번이 $8.8 예산을
넘길 수 있고, 모니터링/알림(ADR-0012)도 이 가드의 **선행 구현**에 종속된다.

## 결정
인메모리 누적기 `createBudgetTracker`(`analyze/budget.ts`)를 두어 `res.usage`(input/output + cache_*)를
Sonnet 4.6 단가($3/$15 per 1M, 캐시 쓰기 1.25×·읽기 0.1×)로 추정·누적한다.
- **pre-flight**: `analyzeUsage`가 호출 전 `canSpend()`를 확인 — 누적 ≥ 상한이면 `client.messages.create`를
  호출하지 않고 `null` 반환(키 없음과 동일한 휴리스틱 폴백, 지출 0).
- **post-call**: `res.usage`가 있으면 refusal/빈응답 early-return **전에** 기록(API가 청구한 토큰 반영).
- **리셋 윈도우 = UTC 달력 월**: 월 경계를 넘으면 누적 0으로 리셋(서킷 닫힘). 시계(`now`) 주입으로 테스트.
- **관측**: `GET /api/usage`가 누적 토큰·추정비용·상한·남은예산·open여부(비민감 집계만) 노출.
- **설정**: `ANTHROPIC_BUDGET_USD`(기본 8) + 단가 변수(`ANALYSIS_INPUT/OUTPUT_USD_PER_MTOK`)를 env zod 검증.
- 인스턴스 스코프(캐시·rate-limit과 동일 패턴) — 서버당 1개, 테스트는 주입으로 격리.

## 대안
- **수동 키 제거 유지** → 사람 개입 의존·사고 방지 불가 → 기각.
- **요청 직렬화로 정확한 상한** → 처리량 저하·복잡도 → YAGNI. 상한을 $8.8보다 낮게(기본 8) 잡아 헤드룸 확보.
- **파일/Supabase 영속 카운터** → MVP 단일 인스턴스엔 과투자 → 후속(인터페이스 유지로 교체 용이).
- **사용량 메트릭 모듈 별도 신설** → 본 트래커가 자체 기록을 포함하므로 중복 → 통합 메트릭(M2)에 흡수.

## 트레이드오프
- (+) 비용 사고를 코드로 봉인. 키 게이트·캐시·폴백 패턴을 그대로 확장(추가 API 호출 0).
- (+) 검색은 끊기지 않음 — 예산 소진 시 휴리스틱 그래프로 자동 폴백(200 유지).
- (−) 인메모리라 프로세스 재시작 시 0으로 리셋 → 잦은 재시작이 반복되면 윈도우 실지출이 상한을 누적 초과 가능. 단일 인스턴스 전제, 한계로 수용.
- (−) 다중 인스턴스 배포 시 카운터가 인스턴스별 독립 → 전체 합산이 상한을 넘을 수 있음. MVP 단일 인스턴스 전제.
- (−) 동시 요청 레이스: record 전 pre-flight 동시 통과로 ~동시성×검색당 비용만큼 초과 가능 → 낮은 기본 상한으로 완화.
- (−) 추정 정확도는 단가에 의존(모델 교체 시 env 동기화 필요) → 불일치는 보수적(상향) 추정.
