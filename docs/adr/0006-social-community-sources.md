# ADR-0006 — 소셜·커뮤니티 소스: 국내 커뮤니티(공식 검색 API) 도입 + SNS 보류

- 상태: Accepted
- 일자: 2026-06-26
- 관련: [ADR-0005](./0005-defer-broad-web-search.md)(광범위 웹검색 보류) 와 동일한 "무료·무카드" 원칙의 연장선

## 맥락
"인스타그램·페이스북·유튜브·X·국내외 커뮤니티 정보도 가져오고 싶다"는 요구를 받고,
7개 후보(YouTube·X·Instagram·Facebook·Reddit·페디버스·국내 커뮤니티)를 2026년 현재
기준 ToS·과금·승인·PIPA로 재검증했다(병렬 리서치 + 적대적 검증).

핵심 발견: **직접 지목된 SNS(인스타·페북·X)가 하필 가장 막힌 곳**이다. 이들은
"내가 소유·인증한 계정"만 공식 API로 열어주고, 제3자(기업·공인)의 공개 콘텐츠를
검색하는 경로는 없거나(페북), 법인 인증·앱심사 게이트(인스타), 유료화(X)됐다.
커뮤니티 직접 크롤링은 ToS·robots·인증벽·PIPA 하드룰 위반이라 선택지가 아니다.

반면 **국내 커뮤니티 콘텐츠는 공식 검색 API로 합법 수집 가능**하다 — 디시·클리앙 등의
공개글 중 검색엔진에 색인된 것은 네이버·카카오(다음) 검색 API를 통해 ToS-clean하게 들어온다.

## 검증 결과 (2026-06)
| 플랫폼 | 판정 | 사유 |
|---|---|---|
| 네이버 blog/cafe/kin | ✅ 무료·합법 | 기존 키 재사용, 공식 공개 API |
| 카카오(다음) web/blog/cafe | ✅ 무료·합법 | REST 키만, 사업자등록 불요. 네이버 미색인 커뮤니티 보완 |
| YouTube Data API v3 | ✅ 무료(키 필요) | 이번 범위 밖(요청 범위가 국내 커뮤니티) — 후속 |
| X(트위터) | 💳 유료만 | 무료 티어 읽기 불가. 읽기 $0.005/post → 월 ~$30~100 |
| Instagram | ⚠️ 고난도 | 법인 Business Verification + 앱심사, 제3자 임의조회는 거절 위험 |
| Reddit | ⚠️ 고난도 | 무료는 비상업 한정, 상업은 계약. PIPA 리스크 높음 |
| Facebook | ❌ 불가 | 공개글 키워드 검색 API 자체가 (무가·유가 모두) 부재 |

## 결정
지금은 **무료·ToS-clean·키만으로 되는 국내 커뮤니티 소스만** 도입한다.
1. **네이버 어댑터 확장**: 기존 webkr/news 에 `blog`·`cafearticle`·`kin` 추가(동일 키).
2. **카카오(다음) 검색 어댑터 신규**: `web`·`blog`·`cafe`(`KAKAO_REST_API_KEY` 미설정 시 자동 비활성).
3. **출처 투명성**: `RawItem.sourceType` 항목별 오버라이드를 도입해 blog→블로그, cafe/kin→커뮤니티로 정확히 배지.
4. **PIPA 경계 가드 신설**: UGC 유입으로 늘어나는 민감정보 노출에 대비, 수집 경계(`normalize`)에서 주민번호·휴대전화 패턴을 마스킹.

SNS는 보류: **X**(유료 — 예산 승인 시 flag+상한 뒤에서만), **Instagram**(법인 인증·앱심사),
**Reddit**(상업 계약). **Facebook**은 도입 불가(어댑터 만들지 않음).

## 대안
- X 즉시 유료 도입 → MVP 단계 카드·과금 리스크, YAGNI 위반으로 기각(ADR-0005와 동일 논리).
- 커뮤니티 직접 스크래핑(디시/클리앙/블라인드 등) → ToS·robots·인증벽 위반, 하드룰로 금지.
- 카카오 생략하고 네이버만 → 네이버 미색인 커뮤니티(일부 디시 등) 사각 발생 → 카카오로 보완.

## 트레이드오프
- (+) 비용 0·카드 0(카카오 키도 무료), 국내 커뮤니티/블로그 커버리지 즉시 확대.
- (+) `SourceAdapter` 추상화로 보류 항목(X·IG·YouTube)은 파일 1개 단위 재도입 가능 → 기회비용 작음.
- (-) X/인스타의 실시간 SNS 여론 커버리지는 당분간 공백(요청 대비 부분 충족).
- (-) UGC 도입으로 PIPA 노출 증가 → 공인 한정·민감정보 마스킹을 소스별로 더 엄격히 적용해야 함.
  (마스킹은 best-effort 경계 가드이며 "공개정보·공인 한정" 정책을 대체하지 않는다.)
- 네이버 일일 쿼터(25,000콜)는 전 엔드포인트 공유 → 엔드포인트 5종 = 쿼리당 5콜. 30분 캐시로 완화.
